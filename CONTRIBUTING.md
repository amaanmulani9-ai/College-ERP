# Contributing to College ERP

First off, thank you for considering contributing to College ERP! It's people like you that make College ERP such a great tool.

## Where do I go from here?

If you've noticed a bug or have a feature request, make sure to check our [Issues](https://github.com/amaanmulani9-ai/College-ERP/issues) page to see if someone else has already created a ticket. If not, go ahead and make one!

## Fork & create a branch

If this is something you think you can fix, then fork College ERP and create a branch with a descriptive name.

A good branch name would be (where issue #325 is the ticket you're working on):

```bash
git checkout -b 325-add-attendance-export
```

## Get the test suite running

Make sure your local environment is correctly setup by following the installation instructions in the `README.md`. Ensure that your code changes don't break existing functionality and that the Django server boots without errors.

## Implement your fix or feature

At this point, you're ready to make your changes. Feel free to ask for help; everyone is a beginner at first.

## Make a Pull Request

When you're ready, create a Pull Request on GitHub. 

- Provide a clear title and description.
- Link the PR to the issue you are solving.
- Wait for the maintainer (Amaan) to review your code.

Thank you for contributing! 🚀
